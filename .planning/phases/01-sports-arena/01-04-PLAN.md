# Plan 01-04: Wire Sports Arena into ExperienceHomePage

## Objective
When `experienceId === 'sports'`, ExperienceHomePage renders the Sports Arena layout instead of the generic grid. Deploy to production.

## Context
- @src/pages/ExperienceHomePage.tsx — the universal experience page
- @src/components/sports/LeagueSelector.tsx — league pills (from 01-02)
- @src/components/sports/MatchDayCards.tsx — fixture cards (from 01-02)
- @src/components/sports/StandingsWidget.tsx — league table (from 01-03)
- @src/hooks/useSportsData.ts — data hook (from 01-03)
- ExperienceHomePage receives `experienceId` from route params
- Existing sections: hero → search → VEE picks → subtab pills → channel grid → social proof
- Sports Arena adds: league selector + match day cards + standings ABOVE the channel grid

## Tasks

### Task 1: Conditional Sports Layout
- **type**: modify
- **file**: `src/pages/ExperienceHomePage.tsx`
- **action**:
  - Import LeagueSelector, MatchDayCards, StandingsWidget, useSportsData (lazy import to avoid loading sports code on other experiences)
  - Detect `isSports = experienceId === 'sports'`
  - If isSports, call `useSportsData()` hook
  - Insert BETWEEN the hero and the subtab pills:
    1. `<LeagueSelector>` — horizontal league pills
    2. `<MatchDayCards>` — fixture cards for active league
    3. `<StandingsWidget>` — compact table (collapsible)
  - These sections only render when `isSports && !isSearching`
  - VEE picks row still shows (AI recommendations complement fixtures)
  - Channel grid still shows below (the actual streams)
  - Sports sections get VEE light beam treatment (warm orange radial gradient behind)
  - Lazy import pattern: `const SportsArena = lazy(() => import('@/components/sports/SportsArena'))` — bundle split so other experiences don't pay the cost
- **verify**: Navigate to /live/sports — see league selector, fixtures, standings above channels

### Task 2: SportsArena Wrapper + Build + Deploy
- **type**: create + deploy
- **file**: `src/components/sports/SportsArena.tsx`
- **action**:
  - Create wrapper component that composes LeagueSelector + MatchDayCards + StandingsWidget + useSportsData
  - Self-contained: handles its own data fetching + state
  - Props: none (it's a standalone section)
  - Layout:
    ```
    <section className="reveal">
      <LeagueSelector ... />
      <MatchDayCards ... />  
      <StandingsWidget ... />  (below cards, collapsible)
    </section>
    ```
  - Wrap in orange light beam gradient (same pattern as VEE Hot)
  - Run `npm run build` — verify zero errors, chunk size reasonable
  - Run `npx vercel --prod` — deploy
  - Commit with descriptive message
- **verify**: 
  - /live/sports shows fixtures + standings
  - Other experience pages (/live/arabic etc.) unaffected
  - Build clean, deployed
